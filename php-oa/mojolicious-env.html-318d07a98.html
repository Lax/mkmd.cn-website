Mojolicious 框架的环境变量列表

扶 凯 2014年08月8日 - 18:31
0

这是 Mojolicious 中找到的变量列表, 有时好要找, 找不到, 我找到了 Wiki 上的, 然后译成中文拿出来了.备查.
注意, 这并不保持下面的列表是全的.

  • MOJO_HOME [Mojo::Home, Mojolicious::Commands (command line opt), Mojolicious::Lite] – 当前工作的主目标
  • MOJO_LOG_LEVEL /debug|info|warn|error|fatal/ [Mojo::Log, Test::Mojo] – 有效的日志级别.
  • MOJO_MODE /'development'|'production'/ [Mojolicious, Mojolicious::Commands (command line opt), Mojolicious::Lite, Mojo::Server::Hypnotoad] – 你写的应用程序工作的模式环境.
  • MOJO_APP [Mojolicious::Commands] -默认的 MOJO_APP 的值是  Mojo::HelloWorld.
  • MOJO_CONFIG [Mojolicious::Plugin::Config] – 全路径的配置文件, 默认是使用的你的应用的 home 目标中的 myapp.conf 文件. 
  • MOJO_NO_DETECT bool [Mojolicious::Commands] – no detect environment (PSGI, CGI, …). 自动检测部署所使用的环境可以用, 可以通过 MOJO_NO_DETECT 来修改它.
  • MOJO_INACTIVITY_TIMEOUT [Mojo::Server::Daemon, Mojolicious::Command::daemon (command line opt), Mojo::UserAgent] – 连接后最大的不活动的时间, 超过会被关闭. 默认 MOJO_INACTIVITY_TIMEOUT 的环境变量是 15 ( UserAgent 中是 20 ). 将该值设置为 0 将允许连接到无限期处于非活动状态.
  • MOJO_LISTEN [Mojo::Server::Daemon, Mojolicious::Command::daemon (command line opt)] – 命令行检查的地址, MOJO_LISTEN 的默认值是 'http://*:3000'.
  • MOJO_REVERSE_PROXY bool [Mojo::Server::Hypnotoad, Mojolicious::Command::daemon (command line opt), Mojo::Message::Request, Mojo::Transaction] – 激活反向代理的支持, 这会让 Mojo 来自动的检查 X-Forwarded-For 和 X-Forwarded-HTTPS headers 来自动的检查, 默认是检查  MOJO_REVERSE_PROXY 环境变量.
  • MOJO_MAX_BUFFER_SIZE [Mojo::Content] – 最大的进行内容 body 解析时的大小, 默认的  MOJO_MAX_BUFFER_SIZE 的值是 262144 (256KB).
  • MOJO_MAX_LEFTOVER_SIZE [Mojo::Content] –  pipelined HTTP 请求时最大的缓冲大小,  默认的 MOJO_MAX_LEFTOVER_SIZE 的值是 262144 (256KB).
  • MOJO_MAX_LINE_SIZE [Mojo::Headers, Mojo::Message] -最大的 header 行的长度, 单件是字节, 默认的这个 MOJO_MAX_LINE_SIZE 值是 10240 (10KB).
  • MOJO_MAX_MESSAGE_SIZE [Mojolicious::Lite, Mojo::Message, Mojolicious::Lite] – 为了保护您免受接收过大的文件默认限制是 10MB (10485760) , 你可以通过 MOJO_MAX_MESSAGE_SIZE 环境变量来调整这个值.
  • MOJO_MAX_WEBSOCKET_SIZE [Mojo::Transaction::WebSocket] – 最大的 WebSocket 消息的大小, 默认是 MOJO_MAX_WEBSOCKET_SIZE 的值是 262144.
  • MOJO_NO_IPV6 bool [Mojo::IOLoop, Mojo::IOLoop::Client, Mojo::IOLoop::Server, Mojo::UserAgent, Mojo::Server::Morbo, Mojo::Server::Daemon, Mojo::Server::Hypnotoad] – IPv6 时禁止使用 use IO::Socket::IP.
  • MOJO_NO_TLS bool [Mojo::IOLoop, Mojo::IOLoop::Client, Mojo::IOLoop::Server, Mojo::UserAgent, Mojo::Server::Morbo, Mojo::Server::Daemon, Mojo::Server::Hypnotoad] – SSL 时禁止使用 IO::Socket::SSL.
  • MOJO_MAX_MEMORY_SIZE [Mojo::Asset::Memory] – 最大只的保持在内存中的字节大小, 超过会自动的使用 Mojo::Asset::File 对象生成临时文件, 默认值是使用 MOJO_MAX_MEMORY_SIZE 环境变量, 或者 262144.
  • MOJO_TMPDIR [Mojo::Asset::File] – 本模块生成文件的临时目录, 默认是从 MOJO_TMPDIR 环境变量中取得或者自动检测.
  • MOJO_REACTOR [Mojo::Reactor] – 检测和加载的最佳的可用的 REACTOR ( 事件驱动 ).这会尝试从 MOJO_REACTOR 环境变量中取指你指定的, 目前有二个 Mojo::Reactor::EV 或者 Mojo::Reactor::Poll.

Mojo::UserAgent (ojo 相同)

  • MOJO_CA_FILE [Mojo::UserAgent] – TLS certificate authority 文件的位置, 默认是从 MOJO_CA_FILE 环境变量中取得. 这会激活主机名验证.
  • MOJO_CERT_FILE [Mojo::UserAgent] – TLS certificate 文件的位置, 默认是从 MOJO_CERT_FILE 环境变量中取得.
  • MOJO_KEY_FILE [Mojo::UserAgent] –  TLS key 文件的路径, 默认是从  MOJO_KEY_FILE 的环境变量中取得.
  • MOJO_CONNECT_TIMEOUT [Mojo::UserAgent] – 建连时的最多的超时时间, 超过会被注销, 默认是  10 或者从环境变量的中的 MOJO_CONNECT_TIMEOUT 取得.
  • MOJO_MAX_REDIRECTS [Mojo::UserAgent] – 最多可以重定向的数量, 超过会失败, 默认这个是是设置为 0 或者从环境变量的  MOJO_MAX_REDIRECTS 中取得.
  • MOJO_REQUEST_TIMEOUT [Mojo::UserAgent] - 建立连接的超时时间, 发送请求和接收整个响应可能需要的时间, 超过会被注消. 默认的 MOJO_REQUEST_TIMEOUT 是使用环境变量的这个值或者 0. 当设置为 0 时会无限斯的等待.  每次 redirect 会自动的重置.
  • MOJO_PROXY [Mojo::UserAgent, ojo] – 检查环境变量中的 HTTP_PROXY, http_proxy, HTTPS_PROXY, https_proxy, NO_PROXY 和 no_proxy 中的代理的信息. 通过这个选项, 可以打开自动的检查代理选项.

Debugging

  • MOJO_EXCEPTION_VERBOSE [Mojo::Exception] - 激活渲染时的详细信息,默认 MOJO_EXCEPTION_VERBOSE 是 0.
  • MOJO_IOLOOP_DEBUG bool [Mojo::IOLoop] – 你可以设置 MOJO_IOLOOP_DEBUG 环境变量来打印高级的事件循环诊断信息到标准错误.
  • MOJO_EVENTEMITTER_DEBUG bool [Mojo::EventEmitter] -你可以设置 MOJO_EVENTEMITTER_DEBUG 环境变量来打印高级的事件调用的诊断信息到标准错误.
  • MOJO_DAEMON_DEBUG bool [Mojo::Server::Daemon] – 你可以设置 MOJO_DAEMON_DEBUG 环境变量来打印高级的守护进程的诊断信息到标准错误.
  • MOJO_TEMPLATE_DEBUG bool [Mojo::Template] - 你可以设置 MOJO_TEMPLATE_DEBUG  环境变量来打印高级的模板的诊断信息到标准错误.
  • MOJO_WEBSOCKET_DEBUG bool [Mojo::Transaction::WebSocket] - 你可以设置 MOJO_WEBSOCKET_DEBUG 环境变量来打印高级的 WebSocket 的诊断信息到标准错误.
  • MOJO_BASE_DEBUG bool [Mojo::Base] - 你可以设置 MOJO_BASE_DEBUG 环境变量来打印高级的 Mojo::Base 的诊断信息到标准错误.
  • MOJO_USERAGENT_DEBUG bool [Mojo::UserAgent] -你可以设置 MOJO_USERAGENT_DEBUG 环境变量来打印高级的 Mojo::UserAgent 的创建连接的请求和响应的诊断信息到标准错误.
  • MOJO_REUSE private [Mojo::IOLoop::Server] – ?
  • MOJO_EXE private [Mojolicious::Lite, Mojolicious::Plugin::Config] – executable
  • MOJO_APP_LOADER private bool [Mojolicious::Commands] – ?
  • PLACK_ENV [Mojolicious::Commands::detect] – detect PSGI (Plack)
  • HARNESS_ACTIVE [Mojolicious::Commands] – test
  • HARNESS_IS_VERBOSE [Test::Mojo] – test

可以通过下面的命令来找出所有的环境变量

for f in $(find $(perl -MMojo -e 'print map { $INC{$_} =~ s/Mojo.pm//; $INC{$_} } grep /Mojo.pm/, keys %INC') -name '*.pm' -path '*ojo*')
do 
  echo $f;
  grep -n "MOJO_" $f;
done
